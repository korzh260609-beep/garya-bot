# Project Structure — Архитектурная карта проекта

## 1. Файловая структура (ветка dev)
/index.js        — главный сервер бота, webhook Telegram  
/db.js           — подключение PostgreSQL  
/sources.js      — слой Sources (скелет)  
/classifier.js   — классификатор задач (скелет)  
/package.json    — зависимости, скрипты  
/WORKFLOW.md     — карта проекта и прогресс разработки  

---

## 2. Основные слои архитектуры

### 2.1 Transport Layer
- Telegram Webhook
- Express сервер
- Обработка и маршрутизация апдейтов

### 2.2 Memory Layer
- Таблица chat_memory
- Сохранение сообщений
- Загрузка последних N сообщений
- ❗ Будущее: долговременная память
- ❗ Будущее: rehydrate (восстановление контекста)

### 2.3 Sources Layer
- Реестр источников (RSS, API, web)
- Скелет функций listActiveSources, ensureDefaultSources
- Source-first подход (позже)

### 2.4 AI Layer
- Системный промпт
- Режимы short / normal / long
- Правило минимально достаточного ответа
- Формирование ответа через OpenAI

### 2.5 Task Engine
- Таблица tasks
- Cron-планировщик (Render)
- Базовые типы задач (manual, price_monitor)
- Robot-layer для бесплатных операций
- Будущее: отчёты, мониторинг, автоматизация
