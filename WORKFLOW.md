# Project Structure — Архитектурная карта проекта

## 1. Файловая структура (ветка dev)
/index.js        — главный сервер бота, Webhook Telegram  
/db.js           — подключение PostgreSQL, работа с chat_memory  
/sources.js      — слой Sources (скелет)  
/classifier.js   — классификатор задач (скелет)  
/package.json    — зависимости и скрипты  
/WORKFLOW.md     — карта проекта и прогресс  

---

## 2. Основные слои архитектуры

### 2.1 Transport Layer
- Telegram Webhook  
- Express сервер  
- Приём и маршрутизация апдейтов  

### 2.2 Memory Layer
- Таблица chat_memory  
- Сохранение сообщений  
- Подгрузка последних N сообщений  
- Долговременная память (в разработке)  
- Rehydrate (в разработке)  

### 2.3 Sources Layer
- Структура источников данных  
- RSS, сайты, API (этап 6–7)  
- Source-first подход  

### 2.4 AI Layer
- System prompt  
- Режимы short / normal / long  
- Правило минимально достаточного ответа  
- Формирование финального ответа  

### 2.5 Task Engine
- Таблица tasks  
- Cron Render  
- Базовые задачи  
- В будущем: отчёты, мониторинг, Automation Logic
