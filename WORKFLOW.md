🔧 Как выглядит твоя рабочая схема
1) Ты создаёшь изменения → копируешь код сюда в чат

Советник:

анализирует

критикует

переписывает

даёт готовые блоки для вставки

2) Ты вставляешь код в GitHub

В ветку main (позже — dev → merge → main)

3) Render автоматически подтягивает деплой

(если включено auto-deploy)

4) Ты тестируешь через Telegram

Команды: /sources, /test_source, /task, /mode, /btc_test_task, /memstats и т.д.

5) Каждое завершённое действие фиксируется в WORKFLOW.md

Чтобы бот в будущем мог опираться на документ (Этап 6+).
